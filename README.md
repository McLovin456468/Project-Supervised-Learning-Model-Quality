# Проект: Персонализация предложений для повышения покупательской активности

## 1. Назначение и цель проекта

Интернет-магазин «...» сталкивается со снижением активности постоянных клиентов. Привлечение новой аудитории неэффективно, так как большинство целевой аудитории уже знает о магазине.

**Цель:** разработать решение для персонализации предложений постоянным клиентам, позволяющее увеличить их покупательскую активность.

## 2. Подход к решению

Руководством утверждён следующий подход:

1. **Маркировка целевого признака** — выделить два уровня финансовой активности постоянных покупателей:
   - «снизилась» — клиент стал покупать меньше товаров;
   - «прежний уровень» — активность не изменилась.

2. **Сбор данных** по клиентам по четырём группам признаков:
   - коммуникация сотрудников с клиентом;
   - продуктовое поведение (какие товары покупает и как часто);
   - покупательское поведение (сколько тратил в магазине);
   - поведение на сайте (количество просмотренных страниц, время на сайте).

3. **Построение модели** для предсказания вероятности снижения покупательской активности в следующие 3 месяца.

4. **Учёт прибыльности клиента** — включить данные о доходе, который каждый покупатель принёс компании за последние 3 месяца.

5. **Сегментация и разработка предложений** — на основе модели и данных о прибыльности выделить сегменты покупателей и сформировать персонализированные предложения.

## 3. Входные данные

| Файл | Содержание |
|------|------------|
| `market_file.csv` | Поведение на сайте, коммуникации, продуктовое поведение |
| `market_money.csv` | Выручка от покупателя по периодам |
| `market_time.csv` | Время на сайте по периодам (минуты) |
| `money.csv` | Среднемесячная прибыль от покупателя за последние 3 месяца |

### 3.1. Описание полей `market_file.csv`

| Поле | Описание |
|------|----------|
| `id` | Номер покупателя |
| `Покупательская активность` | Целевой признак: «снизилась» / «прежний уровень» |
| `Тип сервиса` | Премиум / стандарт (в зависимости от подписки) |
| `Разрешить сообщать` | Согласие на рассылку предложений |
| `Маркет_актив_6_мес` | Среднемесячное число маркетинговых коммуникаций за 6 месяцев |
| `Маркет_актив_тек_мес` | Количество маркетинговых коммуникаций в текущем месяце |
| `Длительность` | Дней с момента регистрации на сайте |
| `Акционные_покупки` | Среднемесячная доля покупок по акции за 6 месяцев |
| `Популярная_категория` | Самая популярная категория товаров за 6 месяцев |
| `Средний_просмотр_категорий_за_визит` | Среднее число просмотренных категорий за визит (последний месяц) |
| `Неоплаченные_продукты_штук_квартал` | Количество неоплаченных товаров в корзине за 3 месяца |
| `Ошибка_сервиса` | Число сбоев при посещении сайта за 3 месяца |
| `Страниц_за_визит` | Среднее количество страниц за визит (последние 3 месяца) |

### 3.2. Описание полей `market_money.csv` и `market_time.csv`

| Поле | Описание |
|------|----------|
| `id` | Номер покупателя |
| `Период` | Период фиксации показателя (например, «текущий_месяц») |
| `Выручка` / `минут` | Значение показателя |

### 3.3. Описание полей `money.csv`

| Поле | Описание |
|------|----------|
| `id` | Номер покупателя |
| `Прибыль` | Среднемесячная прибыль от покупателя за 3 месяца (условные единицы) |

## 4. Этапы выполнения

### 4.1. Загрузка данных
- Загрузить четыре файла.
- Проверить соответствие данных описанию.
- Учесть особенности: разделитель «;», десятичный разделитель «,».

### 4.2. Предобработка данных
- Обработать пропуски, дубликаты, некорректные типы.
- Привести названия столбцов к единому формату (допустимо оставить кириллицу).

### 4.3. Исследовательский анализ данных (EDA)
- Провести анализ каждой таблицы.
- Отобрать клиентов с покупательской активностью не менее 3 месяцев (осуществлявших покупки в этот период).
- Сформулировать выводы.

### 4.4. Объединение таблиц
- Объединить `market_file.csv`, `market_money.csv`, `market_time.csv`.
- Данные о прибыли (`money.csv`) на этапе моделирования не используются, подключаются позже для сегментации.
- Для данных `market_money` и `market_time` выполнить **pivot**: создать отдельные столбцы для каждого периода (текущий месяц, предыдущий и т.д.).

### 4.5. Корреляционный анализ
- Провести корреляционный анализ количественных признаков в итоговой таблице.
- Выявить мультиколлинеарность, при необходимости устранить.

### 4.6. Построение моделей с использованием пайплайнов

#### 4.6.1. Подготовка данных через ColumnTransformer
- Разделить признаки на количественные и категориальные.
- Для категориальных признаков применить **не менее двух кодировщиков** (например, OneHotEncoder, OrdinalEncoder).
- Для количественных признаков применить **не менее двух скейлеров** (например, StandardScaler, MinMaxScaler).
- Для каждой модели можно использовать разные комбинации кодирования и масштабирования.

#### 4.6.2. Обучение и подбор гиперпараметров
Обучить следующие модели:
- `KNeighborsClassifier`
- `DecisionTreeClassifier`
- `LogisticRegression`
- `SVC`

Для каждой модели подобрать **как минимум один гиперпараметр**.

Выбрать и обосновать метрику оценки качества. Использовать эту метрику для подбора гиперпараметров.

#### 4.6.3. Выбор лучшей модели
Способы выбора:
- использовать отдельные пайплайны и подбор гиперпараметров для каждой модели;
- использовать общий пайплайн с инструментом подбора, возвращающим лучшую модель.

### 4.7. Анализ важности признаков
- Оценить важность признаков для лучшей модели.
- Построить график важности с помощью **SHAP**.
- Сделать выводы:
  - какие признаки мало значимы;
  - какие признаки наиболее сильно влияют на целевой признак;
  - как использовать эти наблюдения для моделирования и бизнес-решений.

### 4.8. Сегментация покупателей и персонализация предложений

#### 4.8.1. Сегментация
- Выполнить сегментацию покупателей, используя:
  - результаты модели (вероятность снижения активности);
  - данные о прибыльности (`money.csv`);
  - исходные данные (при необходимости).

#### 4.8.2. Примеры сегментов для анализа (допускается выбрать свой)
- Клиенты с максимальной долей покупок по акции и высокой вероятностью снижения активности.
- Клиенты, покупающие только технику (товары с длинным жизненным циклом).
- Клиенты, покупающие «Товары для себя» или «Товары для детей».
- Клиенты с высокой вероятностью снижения активности и наибольшей прибыльностью.

#### 4.8.3. Исследование выбранного сегмента
- Провести графический и аналитический анализ группы.
- Сформулировать предложения по работе с сегментом для повышения активности.

#### 4.8.4. Выводы по сегментации
- Указать, какой сегмент выбран для исследования.
- Обосновать предложенные меры.

### 4.9. Общий вывод
- Описать решаемую задачу.
- Охарактеризовать исходные данные и выполненную предобработку.
- Изложить подход к поиску лучшей модели.
- Назвать лучшую модель.
- Привести выводы и дополнительные предложения для выбранного сегмента.

## 5. Результаты проекта

- Обученная модель для предсказания вероятности снижения покупательской активности.
- Оценка важности признаков (SHAP).
- Сегментация клиентской базы.
- Персонализированные предложения для целевого сегмента.
- Итоговый отчёт с выводами и рекомендациями.
